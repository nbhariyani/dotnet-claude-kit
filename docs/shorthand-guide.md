# nestjs-claude-kit Quick Reference

## Commands

| Command | Purpose |
|---|---|
| `/nest-init` | Bootstrap a NestJS project or align an existing one |
| `/plan` | Architecture-aware planning |
| `/scaffold` | Generate or structure a feature |
| `/verify` | Run the verification workflow |
| `/tdd` | Test-first workflow |
| `/migrate` | Safe migration workflow |
| `/code-review` | Structured review workflow |
| `/security-scan` | Security-focused review |
| `/de-sloppify` | Cleanup and dead-code reduction |

## Core Skill Areas

| Area | Skills |
|---|---|
| Architecture | `architecture-advisor`, `feature-modules`, `clean-architecture`, `ddd`, `project-structure` |
| API | `controllers`, `api-versioning`, `openapi`, `validation`, `authentication` |
| Persistence | `typeorm`, `prisma`, `migration-workflow`, `configuration` |
| Platform | `docker`, `ci-cd`, `security-scan`, `performance` |
| Testing | `testing`, `verification-loop` |
| Workflow | `autonomous-loops`, `context-discipline`, `workflow-mastery`, `session-management` |

## Core Agents

| Agent | Use For |
|---|---|
| `nestjs-architect` | architecture, structure, module boundaries |
| `api-designer` | controllers, routes, OpenAPI, versioning |
| `orm-specialist` | TypeORM, Prisma, migrations, query review |
| `test-engineer` | Jest, SuperTest, Testcontainers |
| `security-auditor` | auth, validation, secrets, OWASP concerns |
| `code-reviewer` | broad code review and project health |
| `build-error-resolver` | TypeScript and build-error loops |

## MCP-First Shortcuts

| Need | Tool |
|---|---|
| find a symbol | `find_symbol` |
| find callers/usages | `find_references` / `find_callers` |
| inspect exports | `get_public_api` |
| inspect modules | `get_module_graph` |
| check TypeScript errors | `get_diagnostics` |
| find dead code | `find_dead_code` |
| check tests around files | `get_test_coverage_map` |
